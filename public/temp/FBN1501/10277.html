Repetition

Consider the lock below which uses three digits for its code:

Suppose it has a code of \(461\).  Order is important because \(614\) differs from \(461\) and the lock will not open.  It has to be \(4-6-1\).  Therefore, we can say the lock uses a permutation of numbers.

For a lock like this, you can use \(222\) as a code as well.  Therefore, we say that repetition is allowed.  Once the two is used, you can use it again.

For permutations with repetition:

When you have \(\PalBc n\) things to choose from and repetition is allowed,
you have \(\PalBc n\) choices each time.

When choosing \(\PalAb r\) of them, the number of permutations is:
\(\PalBc n \times \PalBc n \times \ldots \times\PalBc n \) (\(\PalAb r\) times)

It is easier to write it by using an exponent of \(\PalAb r\) as follows:
Write \(\PalBc n \times \PalBc n \times \ldots \times\PalBc n \) (\(\PalAb r\) times) as \(\PalBc n^\PalAb r\).

In other words, there are \(\PalBc n\) possibilities for the first choice, then there are \(\PalBc n\) possibilities for the second choice, and so on, multiplying each time.

Consider the lock above again.  For each of the three positions in the code, we can choose from \(\PalBc{10}\) different digits, namely \(0,1,\ldots9\).  Thus there are:

\(\PalBc{10}\times\PalBc{10}\times\PalBc{10}=\PalBc{10}^\PalAb3=1\Tsp000\) permutations.

In this case \(\PalBc n\) (the number of things to choose from) is \(\PalBc{10}\) and \(\PalAb r\) (the number of times you choose a digit) equals \(\PalAb3\).

We see repetition is allowed and order matters.
